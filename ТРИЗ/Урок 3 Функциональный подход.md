Давайте структурируем ваши заметки, завершим упражнения и привяжем это к вашей практике (разработка, аналитика, системы).

## 📐 Структура Урока 3: Функциональный подход

### 1. Базовые определения

| Понятие | Определение | Простыми словами |
|---------|-------------|------------------|
| **Система** | Совокупность взаимодействующих элементов, порождающая **новое свойство** | Целое больше суммы частей (колесо + ось + рама = велосипед, а не куча деталей) |
| **Главная функция** | Модель назначения системы | *Зачем* система существует |
| **Функция** | Описание изменения **изделия** **инструментом** | Что конкретно происходит между двумя элементами |

### 2. Формула функции (Триада)
```
🔧 ИНСТРУМЕНТ + 🎯 ДЕЙСТВИЕ + 📦 ИЗДЕЛИЕ
```

| Компонент | Роль | Вопрос для проверки |
|-----------|------|---------------------|
| **Инструмент** | Носитель функции, совершает работу | *Что воздействует?* |
| **Действие** | Глагол, сущность изменения | *Что именно происходит?* |
| **Изделие** | Объект изменения | *Что меняется?* |

> ⚠️ **Важно:** Инструмент всегда совершает работу. Изделие всегда изменяется (свойство или положение в пространстве).

## 🔍 Анализ вредных функций (важно для ТРИЗ!)

Вы отметили: *«Мы должны пытаться избавиться от вредных функций, действующих на изделие»*

Это ключевой момент! В любой системе есть:

| Тип функции | Пример | Что делать |
|-------------|--------|------------|
| **Полезная** | Дрель → сверлит → отверстие | Сохранять, усиливать |
| **Вредная** | Дрель → нагревает → сверло | Устранять, компенсировать |
| **Недостаточная** | Клей → слабо держит → детали | Усиливать |
| **Избыточная** | Лампа → слишком ярко светит → глаза | Ослаблять |

### Пример из вашей сферы (Python / Lowcode):

| Система | Полезная функция | Вредная функция | Как устранить (ТРИЗ) |
|---------|------------------|-----------------|----------------------|
| **Веб-приложение** | Обрабатывает запросы пользователей | Замедляется при нагрузке | Кэширование (Приём 10: Предварительное действие) |
| **База данных** | Хранит записи | Блокируется при записи | Репликация (Приём 1: Дробление) |
| **API** | Передаёт данные | Раскрывает чувствительные данные | Авторизация (Приём 24: Посредник) |
| **Логирование** | Фиксирует события | Замедляет систему | Асинхронная запись (Приём 21: Проскок) |

## 💡 Главный вывод урока

> **Функция — это всегда изменение.**
> Если нет изменения — нет функции.
> Если не можем назвать глагол — не понимаем суть системы.

Этот навык критически важен для:
- 📋 **Системного анализа** (требования = функции)
- 🐛 **Отладки** (какая функция работает неправильно?)
- 🏗 **Проектирования** (какие функции нужны, какие — вредны?)
- 📄 **Документации** (описываем функции, а не «фичи»)
