1 Запрос EXCH_303_SAMPLE_DOCUMENT_COMMON_EVENT
для выборки из EXCH_COMMON_EVENT
В запросе есть несколько условий:
1.	Выбираем только у которых статус = 0 (это значит что событие не обработано)
2.	Далее выбираем только те события у которых type = 1 INSERT
3.	И exch_event_code = 303 (код нашего потока)
4.	И выборка по времени, только за последние 16 минут, так как идет привязка к периодичности срабатывания задачи

2 Алгоритм
Во-первых есть 3 in / out переменных
- out_status – выходящая переменная, возвращает статус после обработки сообщения
- out_entity_id - возвращает массив ID (пока не ясно почему именно ATOM_ARRAY)
Локальные переменные
•	I_TEST – хранит строку в которую записывается значение параметров из сообщения для отправки, по сути самая главная переменная которая отвечает за сборку всего сообщения для отправки
•	i_search_id – переменная для хранения ссылки на запрос для выборки данных по пробе
•	i_arr - ??? не нашел где используется
•	i_count – счетчик
•	j_count – счетчик для рассчета кол-ва раз обхода массива с результатами
•	common_sapmle_id – хранит список ИД для сообщений / проб 
o	??? тут кстати вопрос, зачем это делать, если можно получить их из запроса i_search_common_events[arr_common_events].exch_event_entity_id
•	i_search_common_events – запрос для выборки событий из очереди
•	arr_common_events – какой-то счетчик
•	exch_event_entity_id – ид атрибут из получаемой записи
•	arr_count – длинна вложенного массива (кол-во результатов для пробы)

Логика выполнения алгоритма:
Сначала надо получить массив с сообщениями из очереди (например их 3)
Затем узнать его длинну
		LENGTH(i_search_common_events) 
Затем проверить пустой или нет массив. Если пусто то и обрабатывать дальше нечего.
LENGTH(i_search_common_events) == 0
Затем вытаскиваем ИД сообщения (он же является и ИД пробы) в отдельную переменную
Выполняем запрос для получения данных по одной пробе
Есть переменная OUT  out_entity_id в которую мы складываем список обработанных сообщений из очереди.
Я добавил еще одну переменную с типом Массив простых типов, и новый блок для складывания туда новых значений.
