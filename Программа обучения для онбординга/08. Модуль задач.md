# Программа обучения: "Работа с задачами"

## **Модуль 1: Создание конфигурации задачи**
- **Цель:** Научить создавать конфигурацию задачи.
- **Темы:**
  ### Шаг 1: Переход к созданию конфигурации
  - Нажатие на кнопку "Создать конфигурацию".
  - Открытие модального окна для создания новой конфигурации.

  ### Шаг 2: Заполнение обязательных полей
  - Выбор типа конфигурации "Задача".
  - Ввод наименования конфигурации.
  - Ввод описания конфигурации.

  ### Шаг 3: Подтверждение создания
  - Нажатие на кнопку "Создать".
  - Система создает конфигурацию типа "Задача".

---

## **Модуль 2: Настройка периода выполнения задачи**
- **Цель:** Научить настраивать период выполнения задачи.
- **Темы:**
  ### Шаг 1: Установка начала периода
  - По умолчанию дата создания конфигурации.
  - Возможность ручного изменения через Дата-пикер.

  ### Шаг 2: Установка конца периода
  - По умолчанию дата создания + 5 лет.
  - Возможность ручного изменения через Дата-пикер.

---

## **Модуль 3: Добавление событий**
- **Цель:** Научить добавлять события для задачи.
- **Темы:**
  ### Событие "По времени"
  - Выбор типа события "По времени".
  - Настройка повторения:
    - Ежедневно.
    - Период (задание даты начала и конца, времени).
    - CRON (использование грамматики CRON).
    CRON-формат используется для задания расписания выполнения задач в модуле задач. Формат включает 5 колонок, определяющих параметры времени выполнения. Секунды в CRON-формате не используются.

  ### Событие "При запуске системы"
  - Выбор типа события "При запуске системы".

  ### Событие "При остановке системы"
  - Выбор типа события "При остановке системы".

  ### Сохранение события
  - Нажатие на кнопку "Добавить".
  - Система отображает добавленное событие.

---

## **Модуль 4: Добавление алгоритмов**
- **Цель:** Научить добавлять алгоритмы в задачу.
- **Темы:**
  ### Шаг 1: Добавление алгоритма
  - Нажатие на кнопку "Добавить алгоритм".
  - Использование поиска для выбора нужного алгоритма.
  - Выбор одного или нескольких алгоритмов.

  ### Шаг 2: Сохранение алгоритма
  - Нажатие на кнопку "Добавить".
  - Система отображает добавленный алгоритм в редакторе задачи.

---

## **Модуль 5: Настройка параметров алгоритма**
- **Цель:** Научить настраивать параметры для алгоритмов.
- **Темы:**
  ### Шаг 1: Добавление источника параметра
  - Выбор типа источника:
    - Константа.
    - Переменная.
    - Скрипт-выражение.
  - Ввод значения для константы (форма зависит от типа параметра).

  ### Шаг 2: Сохранение параметра
  - Нажатие на кнопку "Сохранить".
  - Система отображает добавленный источник параметра.

---

## **Модуль 6: Добавление задачи в расписание**
- **Цель:** Научить публиковать и добавлять задачу в модуль расписания.
- **Темы:**
  ### Шаг 1: Публикация задачи
  - Нажатие на кнопку "Опубликовать".
  - Ввод описания в модальном окне.
  - Отметка чек-бокса.
  - Нажатие на кнопку "Опубликовать".

  ### Шаг 2: Добавление в расписание
  - Нажатие на кнопку "Добавить в расписание".
  - Система выводит уведомление об успешном добавлении.

---

## **Модуль 7: Практические задания**
- **Цель:** Закрепить знания на практике.
- **Задания:**
  1. Создать новую задачу.
  2. Настроить период выполнения задачи.
  3. Добавить событие "По времени" с типом повторения "Ежедневно".
  4. Добавить алгоритм в задачу.
  5. Настроить параметры для алгоритма.
  6. Опубликовать задачу и добавить ее в расписание.
