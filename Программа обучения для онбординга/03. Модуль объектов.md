# Программа обучения: "Работа с атрибутами"

## **Модуль 1: Введение в работу с атрибутами**
- **Цель:** Познакомить сотрудников с основными понятиями и назначением атрибутов.
- **Темы:**
  - Что такое атрибуты и их роль в системе.
  - Бизнес-проектирование артефактов (атрибуты и объекты).
  - Основные шаги для работы с атрибутами.

---

## **Модуль 2: Создание атрибута**
- **Цель:** Научить создавать новые атрибуты в системе.
- **Темы:**
  ### Шаг 1: Переход к созданию конфигурации
  - Нажатие на кнопку "Создать конфигурацию".
  - Открытие модального окна для создания новой конфигурации.

  ### Шаг 2: Заполнение обязательных полей
  - Выбор типа конфигурации "Атрибут".
  - Ввод наименования атрибута.
  - Ввод описания атрибута.

  ### Шаг 3: Проверка уникальности атрибута
  - Убедиться, что данный атрибут отсутствует в системе.
  - Возможность переиспользования существующих атрибутов.

  ### Шаг 4: Подтверждение создания
  - Нажатие на кнопку "Создать".
  - Система создает конфигурацию типа "Атрибут".

---

## **Модуль 3: Настройка параметров атрибута**
- **Цель:** Научить настраивать технические параметры атрибута.
- **Темы:**
  ### Шаг 1: Заполнение технического имени
  - Требования к техническому имени:
    - Длина: от 1 до 30 символов.
    - Разрешенные символы: 'A'-'Z', 'a'-'z', '0'-'9', '_'.
    - Не должно начинаться с цифры.
  - Важно: Техническое имя нельзя редактировать после публикации.

  ### Шаг 2: Выбор типа данных
  - Обзор доступных типов данных.
  - Особенности выбора типа данных "Идентификатор":
    - Выбор конфигурации объекта, на который ссылается атрибут.
    - Выбор атрибута связанного объекта.

  ### Шаг 3: Сохранение изменений
  - Нажатие на кнопку "Сохранить".
  - Система сохраняет конфигурацию атрибута.

---

## **Модуль 4: Просмотр и управление атрибутами**
- **Цель:** Научить просматривать и управлять списком созданных атрибутов.
- **Темы:**
  ### Просмотр списка атрибутов
  - Отображение списка атрибутов в выбранной группе.
  - Информация о каждом атрибуте:
    - Наименование.
    - Тип данных.
    - Описание.

  ### Редактирование атрибутов
  - Изменение описания атрибута.
  - Важно: Техническое имя нельзя редактировать после публикации.

  ### Удаление или деактивация атрибутов
  - Процесс удаления атрибута (если это возможно).
  - Деактивация атрибута для исключения из использования.

---

## **Модуль 5: Практические задания**
- **Цель:** Закрепить знания на практике.
- **Задания:**
  1. Создать новый атрибут с уникальным наименованием и описанием.
  2. Заполнить техническое имя и выбрать тип данных.
  3. Создать атрибут с типом данных "String".
  4. Просмотреть список созданных атрибутов.
  5. Попробовать отредактировать описание созданного атрибута.
  6. Убедиться, что техническое имя нельзя изменить после публикации.

# Программа обучения: "Работа со справочниками"

## **Модуль 1: Введение в работу со справочниками**
- **Цель:** Познакомить сотрудников с основными понятиями и назначением справочников.
- **Темы:**
  - Что такое справочники и их роль в системе.
  - Бизнес-проектирование атрибутов и объектов для справочников.
  - Основные шаги для работы со справочниками.

---

## **Модуль 2: Создание объекта для справочника**
- **Цель:** Научить создавать объекты, которые будут использоваться как справочники.
- **Темы:**
  ### Шаг 1: Переход к созданию конфигурации
  - Нажатие на кнопку "Создать конфигурацию".
  - Открытие модального окна для создания новой конфигурации.

  ### Шаг 2: Заполнение обязательных полей
  - Выбор типа конфигурации "Объект".
  - Ввод наименования объекта.
  - Ввод описания объекта.

  ### Шаг 3: Настройка параметров объекта
  - Выбор типа отношений: "Корневой".
  - Выбор категории данных: "Локальные справочные".
  - Создание таблицы для объекта:
    - Заполнение технического имени таблицы (формат `name_of_table`).
    - Требования к имени таблицы:
      - Длина: от 3 до 30 символов.
      - Строчные латинские буквы, нижние подчеркивания вместо пробелов.
    - Нажатие на кнопку "Создать".

  ### Шаг 4: Подтверждение создания
  - Нажатие на кнопку "Создать".
  - Система создает конфигурацию типа "Объект".

---

## **Модуль 3: Добавление атрибутов в объект**
- **Цель:** Научить добавлять атрибуты в объект для формирования справочника.
- **Темы:**
  ### Шаг 1: Добавление атрибутов
  - В таблице "Атрибуты" нажать кнопку "Добавить атрибут".
  - Выбор атрибутов из списка (можно выбрать несколько).
  - Поиск атрибутов по наименованию.
  - Если атрибута нет в списке, его необходимо создать (см. модуль "Работа с атрибутами").

  ### Шаг 2: Проверка добавленных атрибутов
  - Проверка корректности добавленных атрибутов:
    - Наименование.
    - Техническое имя.
    - Тип данных.

  ### Шаг 3: Настройка заголовка справочника
  - В поле "Заголовок" выбрать атрибут, который будет отображаться в интерфейсе системы как заголовок справочника.
  - По умолчанию используется ID.

---

## **Модуль 4: Наполнение справочника**
- **Цель:** Научить добавлять, редактировать и управлять экземплярами справочника.
- **Темы:**
  ### Шаг 1: Переход к справочнику
  - Нажатие на кнопку "Перейти к справочнику".
  - Открытие модального окна "Справочник".

  ### Шаг 2: Добавление экземпляра
  - Нажатие на кнопку "Добавить экземпляр".
  - Заполнение полей экземпляра (максимальная длина строки — 255 символов).
  - Нажатие на кнопку "Сохранить".

  ### Шаг 3: Управление экземплярами
  - Редактирование экземпляра:
    - Нажатие на кнопку "Редактировать экземпляр".
    - Внесение изменений и сохранение.
  - Удаление экземпляра:
    - Нажатие на кнопку "Удалить экземпляр".
  - Просмотр удаленных экземпляров:
    - Нажатие на кнопку "Показать удаленные экземпляры".
  - Восстановление удаленного экземпляра:
    - Нажатие на кнопку "Восстановить экземпляр".

---

## **Модуль 5: Сохранение и публикация**
- **Цель:** Научить завершать работу с конфигурацией и сохранять изменения.
- **Темы:**
  - Нажатие на кнопку "Сохранить" в шапке страницы.
  - Система сохраняет конфигурацию объекта.

---

## **Модуль 6: Практические задания**
- **Цель:** Закрепить знания на практике.
- **Задания:**
  1. Создать новый объект для справочника.
  2. Добавить атрибуты в объект.
  3. Настроить заголовок справочника.
  4. Добавить несколько экземпляров в справочник.
  5. Отредактировать один из экземпляров.
  6. Удалить экземпляр и восстановить его.

---
