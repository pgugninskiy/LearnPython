# 🛒 Домашнее задание: Шаблоны и формы в Django

## 🎯 Цель работы
Закрепить навыки работы с HTML-шаблонами, передачи данных из контроллеров (views) в шаблоны и реализации форм для пользовательского ввода.

## 🏆 Результат
Динамическое веб-приложение для интернет-магазина с отображением данных через шаблоны и возможностью добавления и редактирования товаров через формы.

### 1. Создание шаблонов
- [x] **Базовый шаблон:** Настроен файл `base.html` с использованием тегов `{% block %}` и `{% extends %}` для наследования структуры.
- [x] **Список товаров:** Создан `product_list.html`, где отображаются название, описание и цена каждого товара.
- [x] **Детали товара:** Настроен `product_detail.html` с выводом всех данных о конкретном товаре.

### 2. Создание форм
- [x] **Форма добавления:** Настроена форма `ProductForm` (на основе `ModelForm`) для создания нового товара.
- [x] **Форма редактирования:** Реализована возможность редактирования существующего товара через ту же форму с передачей параметра `instance`.

### 3. Связь с шаблонами и контроллерами
- [x] **Обработка ввода:** В контроллерах (`views.py`) реализована обработка `GET` (отображение формы) и `POST` (сохранение данных) запросов.

  - `Опубликовать выбранные товары` (массовое изменение статуса).
  - `Увеличить цену на 10%` (массовое изменение цены).

## 🚀 Инструкция по запуску и проверке

### 1. Подготовка окружения
Убедитесь, что виртуальное окружение активировано:
```bash
.venv\Scripts\activate   # Для Windows
```

### 2. Применение миграций
Создайте таблицу для модели `Product` в базе данных:
```bash
python manage.py makemigrations
python manage.py migrate
```

### 3. Создание администратора
Создайте суперпользователя для доступа к админ-панели:
```bash
python manage.py createsuperuser
```
*(Следуйте инструкциям в терминале для ввода логина и пароля)*

### 4. Запуск сервера
```bash
python manage.py runserver 8001
```
